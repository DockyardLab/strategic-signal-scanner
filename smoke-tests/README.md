# Smoke Tests and Development Flow

This folder keeps the development / smoke test notes in one place so the repo root stays clean.

## What is here

- `signal_scanner_dev_test_flow.md`: the step-by-step development and smoke test recap
- `signal_replay.md`: local replay notes

## Why this exists

The public repo keeps implementation code at the top level and moves longer process notes into this folder.

That makes the repo easier to scan for:

- new contributors
- agents
- people who only want the runnable pipeline
